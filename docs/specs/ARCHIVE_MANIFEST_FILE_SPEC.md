# Archive Manifest File Specification

## Purpose
Defines the structure and integrity requirements for the `.sgarch` mission manifest files generated by LARI-ArcHive™.

## Current Truth-State
Manifests are the primary output of a SCIMEGA™ mission. They capture the "truth-state" of the mission for later replay, audit, and training.

## Canon Position
A manifest is a multi-section document containing:
- **Header**: Versioning, mission ID, and operator ARC identity.
- **Environment Snapshot**: Weather data and location context.
- **Flight Record**: High-fidelity telemetry frames (simulated or dry-link).
- **Event Ledger**: Discrete lifecycle events (takeoff, mode change, BANE gate, anchor hold).
- **Audit Spine**: Cryptographic hashes of all preceding sections.

## Implementation Status
- **LARI-ArcHive™ Packaging**: Fully implemented for simulation/dry-link missions.
- **Replay Support**: Manifests can be re-imported into the SCIMEGA™ UI for timeline review.
- **Integrity Check**: Basic hash verification on import.

## Known Limitations
- **File Size**: Large missions generate substantial manifests; currently lacks compression.
- **Metadata-Only**: In v0.1.2, telemetry data is marked as "simulated" or "dry-link" only.

## Next Required Work
- **Section Signing**: Implement individual cryptographic signatures for the Event Ledger to prevent partial manifest tampering.
- **Export Filters**: Allow operators to export sanitized manifests (e.g., removing private location data).
