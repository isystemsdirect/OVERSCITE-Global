# SCINGULAR Global Antigravity Rule Reference

> This file links the OVERSCITE workspace to the shared SCINGULAR global Antigravity rule base.
> The global rule set is applied via the `user_global` rule in Antigravity workspace settings.

## Authority Source

The global rule base is maintained in the Antigravity configuration layer and applies across all SCINGULAR workspaces. It governs:

- Development posture (conservative, auditable, reversible)
- Canon protection (no casual renaming, collapsing, or reinterpretation)
- Required architectural distinctions (Scing ≠ Scingular, ARC ≠ OVERSCITE, SRT ≠ general capture)
- Human-anchored governance (augment, never replace)
- Change scope discipline (narrow diffs, reviewable boundaries)
- Security and privacy posture (secure by design, no hardcoded secrets)

## Core Non-Negotiables

These principles are absolute and override all workspace-local preferences:

1. **Human authority is final** — Director and Scing authority outrank all autonomous logic.
2. **No layer collapse** — Protocol ≠ Product. Interface ≠ System. Identity ≠ Workspace.
3. **Archive before mutate** — Preserve lineage of prior intent before replacing structures.
4. **Stabilize before integrate** — Validate local correctness before broad propagation.
5. **Preserve decision lineage** — Never optimize away traceability.
6. **Do not silently broaden authority, autonomy, or permissions.**
7. **Do not rename canonical system terms casually.**
8. **Git is the audit spine** — All changes must remain attributable and reviewable.
9. **Antigravity is an execution instrument, not a sovereign architect.**

## Authority Precedence

In order of descending authority:

1. Director instruction
2. Scing (CDG)
3. Antigravity Global Rules (`user_global`)
4. Workspace Rules (`10-workspace-rules.md`)
5. Workflows
6. Task prompts
7. Git / Runtime / Validation
