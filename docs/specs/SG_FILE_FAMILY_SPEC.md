# .sg* File Family Specification

## Purpose
Defines the directory structure, schema, and purpose of the specialized SCINGULAR™ (.sg*) file family used for data persistence, archival, and intelligence exchange.

## Current Truth-State
The ecosystem utilizes several proprietary file extensions to distinguish governed artifacts from generic data. These files are typically JSON-wrapped or binary-wrapped with cryptographic headers and mandatory metadata envelopes.

## Canon Position
### Core Ecosystem Extensions
- **.sgcb (Command Block)**: Governed instruction sets cleared by BANE/TEON. Internal state is JSON/Binary; external expression is often YAML.
- **.sgr (Sovereign Report)**: The unified container for authenticated DocuSCRIBE™ reports. Replaces or wraps constituent `.sgtx`/.sggr/.sgta artifacts.
- **.sgi (Identity Capsule)**: Secure container for ARC identity state, certifications, and authority keys.
- **.sge (Encrypted Envelope)**: Governed transport container for sensitive data payloads.
- **.sgx (Exchange Packet)**: Governed import/export container for knowledge graphs and cross-workspace intelligence.
- **.sgarch (ArcHive™ Manifest)**: The canonical mission manifest generated by the ArcHive™ engine.

### DocuSCRIBE™ Component Extensions
- **.sgtx**: Governed Text Artifact.
- **.sggr**: Governed Graph Artifact.
- **.sgta**: Governed Table Artifact.
- **.sgpm**: Governed Project Manager Artifact.
- **.sgpl**: Governed Project Plan Artifact.

## Technical Requirements
### MIME Mappings
- `application/x-scingular-cb` (.sgcb)
- `application/x-scingular-report` (.sgr)
- `application/x-scingular-identity` (.sgi)
- `application/x-scingular-envelope` (.sge)
- `application/x-scingular-exchange` (.sgx)
- `application/x-scingular-archive` (.sgarch)

### Handler Requirements
Every .sg* extension must have a corresponding:
- **Parser**: Translates raw file data into internal memory structures.
- **Serializer**: Translates internal structures into the canonical file representation.
- **Validator**: Enforces schema integrity, version compatibility, and cryptographic seal verification.

### Command Block Duality
- **YAML Expression**: The human-readable and authorable form of a Command Block used for rehearsal and review.
- **.sgcb Internal State**: The immutable, sealed, and validated representation used for system-to-system exchange and (future) execution.

## Implementation Status
- **.sgtx / .sggr / .sgta**: **IMPLEMENTED**. See `src/lib/docuscribe/sovereign-file-classes.ts`.
- **.sgarch**: **IMPLEMENTED**. See `src/lib/scimega/archive/archive-assembler.ts`.
- **.sgcb**: **PARTIAL**. Structure modeled; terminal simulation active. No binary handler.
- **.sgr / .sgi / .sge / .sgx**: **SPEC_DEFINED / MISSING**. Handlers not yet implemented in v0.1.2.

## Known Limitations
- **Binary Compatibility**: Files are currently plain-text JSON; no optimized binary representation exists.
- **Parser Availability**: Handlers are restricted to the TypeScript/ScingOS environment.
