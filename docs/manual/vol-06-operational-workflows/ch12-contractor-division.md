## 12.1 Why Contractor Governance Matters

In construction and inspection work, the relationship between project owners, prime contractors, and subcontractors is governed by a web of licensing requirements, insurance obligations, registration mandates, and competency certifications. When something goes wrong on a project — a worker is injured, a code is violated, a defective installation fails — the first question is always: "Who was responsible, and were they properly qualified?"

Traditional project management software tracks contractor names and contact information. It does not track whether their licenses are current, whether their insurance is adequate, whether their workers are properly trained, or whether the oversight chain between prime and sub is documented.

OVERSCITE's **Contractor Division** addresses this gap. It provides a governed system for tracking contractor parties, their qualifications, their compliance status, and the hierarchical oversight relationships between them. Its truth-state is **[FUNCTIONAL MVP]**.

## 12.2 Contractor Division Components

The Contractor Division (`/contractor`) is built from seven integrated components, each handling a specific aspect of contractor governance:

### Party Intake

The entry point for adding a contractor to the system. Party Intake captures:

| Field | Description |
|-------|-------------|
| `company_or_person_name` | Legal entity or individual name |
| `role_type` | Classification: prime_contractor, subcontractor, supplier, consultant |
| `prime_or_sub` | Binary designation: prime (contracts with owner) or sub (contracts with prime) |
| `direct_to_owner` | Whether this party has a direct contractual relationship with the property owner |
| `trade_class` | Trades performed (General, Electrical, Plumbing, HVAC, Roofing, etc.) |
| `self_perform_allowed` | Whether this party performs work directly or only manages subcontractors |

Party Intake is the first gate — it establishes who the contractor is and what role they play in the project hierarchy.

### Governance Setup

After a party is entered, Governance Setup defines the compliance requirements that apply to this specific contractor based on their role, trade, and jurisdiction. This includes:

- **Required licenses** — state contractor license, trade-specific licenses, business licenses
- **Required insurance** — general liability, workers' compensation, professional liability
- **Required registrations** — state registrations, apprenticeship program compliance, safety certifications
- **Bonding requirements** — performance bonds, payment bonds where applicable
- **Competency certifications** — OSHA training, manufacturer-specific certifications

Governance Setup does not determine these requirements arbitrarily. It maps the contractor's profile against jurisdictional requirements to produce a compliance checklist specific to their situation.

### Verification State Banner

Each contractor party displays a **Verification State Banner** — a prominent status indicator showing their current compliance position:

| Status | Meaning |
|--------|---------|
| `verified_active` | All required documentation is on file and current |
| `verified_registered_only` | Registration confirmed, but additional documentation pending |
| `unverified` | Documentation has not been submitted or reviewed |
| `expired` | Previously verified, but one or more credentials have expired |
| `suspended` | Verification revoked due to compliance failure |

The Verification State Banner is always visible when reviewing a contractor's profile. It provides an at-a-glance summary of whether this party's paperwork is in order.

### Documentation Upload Control

The mechanism for uploading and managing compliance documentation:
- License certificates (with expiration dates)
- Insurance certificates of coverage
- Safety training records
- Workers' compensation certificates
- Bond confirmations

Each uploaded document is timestamped, attributed to the uploader, and linked to the specific compliance requirement it satisfies. The system tracks expiration dates and will flag documentation that is approaching or past its expiration.

### Proposal Drafting

When engaging a new contractor, OVERSCITE can assist with proposal and scope-of-work documentation. Proposal Drafting provides:

- Structured proposal templates aligned to trade and project type
- Scope definition tools that map proposed work to building systems
- Cost estimation frameworks
- Timeline and dependency management

All proposal content generated by the system is marked as [CANDIDATE] — it is a starting point for human review, not a binding document. The operator reviews, modifies, and approves the proposal before it is issued.

### Oversight Chain

The Oversight Chain is a visual representation of the hierarchical relationships between project parties. It renders as a tree diagram showing:

- The **prime contractor** at the top, with their verification status and compliance indicators
- **Subcontractors** below, connected by governance links showing their relationship to the prime
- Each node displays the party's name, role type, verification status, license requirements, and registration requirements

```
          ┌────────────────────────┐
          │   Prime Contractor     │
          │   ☑ VERIFIED ACTIVE    │
          │   License: Required    │
          │   Registration: Yes    │
          └──────────┬─────────────┘
                     │
          ┌──────────┴──────────┐
          │                     │
    ┌─────┴────┐         ┌─────┴────┐
    │  Sub #1  │         │  Sub #2  │
    │  ☑ REG   │         │  ⚠ UNVER │
    │  Elec.   │         │  Plumb.  │
    └──────────┘         └──────────┘
```

**Critical principle**: The Oversight Chain enforces a fundamental rule — **compliance status is not inherited.** A subcontractor is not compliant just because the prime contractor is compliant. Each node in the chain is an independent verification locus. Every party must maintain its own governed verification evidence.

This principle is stated explicitly in the Oversight Chain's interface: "Each node represents an independent verification locus. Compliance status is not inherited from upstream Prime contractors; all participants must maintain their own governed verification evidence. No aggregate compliance scoring is permitted."

### Scing Draft Wizard

The Scing Draft Wizard button appears in the Contractor Division to provide Scing-assisted document generation. When activated, it opens the Scing Panel with the contractor context pre-loaded, allowing the operator to request:

- "Draft a subcontractor agreement for ElectriSeal Specialties"
- "Generate a compliance checklist for a plumbing subcontractor in California"
- "Create a scope addendum for the roof replacement phase"

All Scing-generated outputs are [CANDIDATE] and require human review before they become operational.

## 12.3 The Verification Lifecycle

A contractor party progresses through verification states based on documentation:

```
Party Created → unverified
    │
    ▼ (Documentation submitted)
Partial Review → verified_registered_only
    │
    ▼ (All requirements satisfied)
Full Compliance → verified_active
    │
    ▼ (Credential expires)
Expiration Alert → expired
    │
    ▼ (Renewed documentation uploaded)
Re-verification → verified_active
```

The system does not automatically verify documents. It tracks what has been submitted, maps it against what is required, and surfaces the compliance gaps. A human reviewer confirms that the submitted documentation actually satisfies the requirement. This is another instance of non-autonomous design — the system manages the data, but the human makes the compliance determination.

## 12.4 Chapter Summary

The Contractor Division provides governed lifecycle management for all project parties — from initial intake through documentation, verification, oversight chain visualization, and ongoing compliance monitoring. Each contractor is an independent compliance node. No party inherits verified status from another. And all compliance determinations require human review.

This system protects project owners, inspectors, and contractors by creating a documented, auditable record of who was qualified, when, and by what evidence.
